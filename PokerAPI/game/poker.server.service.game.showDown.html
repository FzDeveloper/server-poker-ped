poker.server.service.game.showDown

Retourne le(s) vainqueur(s) du(es) pot(s) suite à l'abbattage des cartes

REST Request

http://localhost:8080/PokerServer/rest/game/showDown/{consumerKey}/{signature}

Authentification

Cette méthode exige une authentification.

Arguments

consumerKey: String (Obligatoire)
Your API application key. See here for more details.
signature: String (Obligatoire)
Tha signature generated from the String
 (secret+"&consumerKey&"+consumerKey+"&token&"+token+"&tableName&"+tableName)

Arguments pour la génération de la signature (à faire côté client)

secret: String (Obligatoire)
le code secret délivré au client lors de la génération du consumerKey
consumerKey: String (Obligatoire)
décrit plus haut
token: String (Obligatoire)
Le token d'accès qui a été envoyé à l'application
tableName: String (Obligatoire)
Le nom de la table sur laquelle se passe le showDown

Returns

playerCards: List<JSONObject> {name: String, cards: List<int>>}
Each name of a player and the list of two cards for each player
winners : List<JSONObject> {winner: String, int hand, cards: List<int>>}
The name of the winner and the list of cards 

Exemple de réponse 

{
 "stat":"ok",
 "playerCards" [ {"John", cards [23;51] ; "Alex", cards [3;33]} ; "Max", cards [3;1] ; "Nolan", cards [2;11] ; "Alice", cards [2;5] },
 "winners" [ {"John", cards [23;51;4;47;5]} ]
}

Codes d'erreur

11: NOT_END_ROUND_POKER
This is not the end of the round
12: NOT_ENOUGH_FLIPED_CARDS
There isn't enough cards to do showDown
13: NO_PLAYER_IN_GAME
There is no players in game
17: UNKNOWN_CONSUMER_KEY
Unknown consumer key
23: INVALID_SIGNATURE
Invalid signature
24: UNKNOWN_ACCESS_TOKEN
The access token is not valid
25: INCOMPATIBLE_ACCESS_TOKEN_CONSUMER
Incompatible access token and consumer key
