poker.server.service.game.authenticate

Connecte le joueur au serveur

REST Request

http://localhost:8080/PokerServer/rest/game/authenticate/{consumerKey}/{signature}

Authentification

Cette méthode exige une authentification.

Arguments

consumerKey: String (Obligatoire)
Your API application key. See here for more details.
signature: String (Obligatoire)
Tha signature generated from the String
 (secret+"&consumerKey&"+consumerKey+"&token&"+token+"&playerName&"+playerName+"&password&"+password)

Arguments pour la génération de la signature (à faire côté client)

secret: String (Obligatoire)
le code secret délivré au client lors de la génération du consumerKey
consumerKey: String (Obligatoire)
décrit plus haut
token: String (Obligatoire)
Le token d'accès qui a été envoyé à l'application
playerName: String (Obligatoire)
Le nom du compte du joueur
password: String (Obligatoire)
Le mot de passe du compte du joueur

Returns

alreadyConnected: boolean
L'état de joueur après déconnexion (si vrai consulter le tableName)
tableName: String
The name of the table where the player is connected

Exemple de réponse 

{
 "stat":"ok", 
 "alreadyConnected":true
 "tableName":"LabriTexasHoldem_55b09952-1ae3-4585-9f2f-d4ad69ff5154"}
}

Codes d'erreur

1: NOT_CORRECT_PASSWORD
The password of the player is incorrect
2: PLAYER_INGAME
The player is already connected to a current game
9: NOT_ENOUGH_MONEY
The player has not enough money to play a game
17: UNKNOWN_CONSUMER_KEY
Unknown consumer key
23: INVALID_SIGNATURE
Invalid signature
24: UNKNOWN_ACCESS_TOKEN
The access token is not valid
25: INCOMPATIBLE_ACCESS_TOKEN_CONSUMER
Incompatible access token and consumer key
