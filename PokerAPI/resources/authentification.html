Authentification de l'utilisateur

De nombreuses méthodes d'API de Poker exigent que l'utilisateur se connecte. La seule authentification des utilisateurs à être utilisée devrait être celle basée sur la spécification OAuth, la norme du secteur. Avec la norme OAuth, vos applications permettent à vos utilisateurs de se connecter de manière sécurisée à leurs comptes Poker.

Utilisation d'OAuth avec Poker

Qu'est-ce qu'"OAuth" ?

OAuth est un protocole ouvert, simple et sécurisé qui permet aux applications d'authentifier les utilisateurs et d'interagir avec Poker en leur nom. Les informations de l'utilisateur final sont transférées en toute sécurité sans révéler l'identité de l'utilisateur. Chez Poker, OAuth sert à vérifier qu'une application qui tente d'interagir avec Poker en votre nom ne le fait qu'en vertu des autorisations que vous lui avez préalablement accordées.

Le présent document est un complément à la spécification OAuth Core 1.0 Revision A et détaille le fonctionnement du processus d'autorisation avec Poker.

Le plus simple pour débuter avec Oauth, c'est d'avoir recours à une bibliothèque avec votre langage de programmation. Pour trouver une bibliothèque correspondant à votre langage de programmation, reportez-vous à la page Code OAuth.

Les 3 étapes du flux OAuth :

  1. Obtenir un jeton de requête
  2. Obtenir l'autorisation de l'utilisateur
  3. Échanger le jeton de requête contre un jeton d'accès

Flux d'autorisation OAuth de Poker

Demandes de signature

Vous devez signer toutes les requêtes dans l'API Poker. Actuellement, Poker prend uniquement en charge le chiffrement de signature AES (SHA1PRNG)

Vous devez d'abord créer une chaîne de base à partir de votre requête. La chaîne de base est élaborée par concaténation de tout les paramètres demandés par un service (voir API Poker), et séparés par "&".

Par exemple, avec l'URL suivante :

http://server:port/PokerServer/rest/game/authenticate/
4996a7d8-92d9-4f17-9853-fbb02ca950ac/
PSDVNZI0303LDL2ELDL2244DKFJ

Aurait une chaîne de base du type :

secret+"&consumerKey&"+consumerKey+"&token&"+token+"&playerName&"+playerName+"&password&"+password

Ce qui générerait la signature suivante :

PSDVNZI0303LDL2ELDL2244DKFJ (qui est donné en paramètre d'une requête afin de récupérer côté serveur le "token", "name",... et traiter la requête.

Obtention d'un jeton de requête

URL de jeton de requête:

http://server:port/PokerServer/rest/auth/requestToken/

Tout d'abord, l'autorisation pour un utilisateur passe par l'obtention d'un jeton de requête avec votre Clé client. Ceci est un jeton temporaire qui servira à authentifier l'utilisateur auprès de votre application. Ce jeton, couplé au jeton secret, est ensuite échangé contre un jeton d'accès.

http://server:port/PokerServer/rest/auth/requestToken/
4996a7d8-92d9-4f17-9853-fbb02ca950ac/ (consumerKey)
DOCPVVKJFKJDFKJE42223PCCNXBS33234455 (signature)

Poker retourne une réponse comparable à la suivante :

{
 "stat":"ok",
 "oauthToken":"90ecb0b7-52aa-4167-b073-e2ebbe68f376",
 "oauthTokenSecret":"77c89332-46f9-410a-af67-fc963d13bc26"
}

Obtention de l'autorisation de l'utilisateur

URL d'autorisation de l'utilisateur:

http://server:port/PokerServer/rest/auth/authorizeRequestToken/

Une fois qu'elle a obtenu le jeton de requête, votre application doit présenter la page d'autorisation de Poker à l'utilisateur qui se voit demander d'accorder à votre application l'autorisation d'accéder à ses données. La page d'autorisation présente à l'utilisateur la liste des autorisations que votre application demande, selon les paramètres indiqués dans le flux d'authentification.

Voici un exemple d'URL d'autorisation :

http://server:port/PokerServer/rest/auth/authorizeRequestToken/
72157626737672178-022bbd2f4c2f3432/ (request token)

Voici un exemple de rappel après autorisation :

{
 "stat":"ok"
 "verifier":"4ff6a7d8-00f9-4f17-fdfd53-vvb02cadf3440ac"
}

Échange du jeton de requête contre un jeton d'accès

URL de jeton d'accès:

http://server:port/PokerServer/rest/auth/getToken/

Une fois que l'utilisateur a autorisé votre application, vous pouvez échanger le jeton de requête approuvé contre un jeton d'accès. Ce jeton d'accès est à stocker par votre application et à utiliser pour lancer des requêtes autorisées vers Poker.

Voici un exemple de requête pour un jeton d'accès :

http://server:port/PokerServer/rest/auth/getToken/
4996a7d8-92d9-4f17-9853-fbb02ca950ac/ {consumerKey}
4996a7d8-92d9-4f17-fdfdf-pfdpofdpo344 {requestToken}

Poker retourne une réponse comparable à la suivante :

{
 "stat":"ok",
 "oauthToken":"90ecb0b7-52aa-4167-b073-e2ebbe68f376",
 "oauthTokenSecret":"77c89332-46f9-410a-af67-fc963d13bc26"
}

Appel de l'API Poker avec OAuth

Une fois qu'un jeton d'accès a été accordé à votre application, vous pouvez lancer des requêtes authentifiées vers l'API Poker. Poker exige le chiffrement AES (SHA1PRNG) car toutes les requêtes sont effectuées de manière non sécurisée via HTTP.

Voici un exemple de requête lancée vers l'API poker.server.service.player.fold :

http://server:port/PokerServer/rest/player/fold/
3030445-dffldflfdl-3d33-dfmfmdf-fmldfdmf3ds22/ {consumerKey}
SPDPSDP303303944XCXCXCKQQSD2440PWWMQK330 {signature}

Poker retourne une réponse comparable à la suivante :

{
 "stat":"ok"
}
